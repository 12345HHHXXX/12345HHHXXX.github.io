二、枚举类型应用场景

发表于2026-06-23

枚举（Enum）是Java中的一种特殊类型，用于定义固定数量的常量集合。以下是枚举的三个常见应用场景：

场景1：状态/类型定义（最常用）

应用场景

在实际项目中，经常需要定义一些固定的状态，如订单状态、用户状态、支付状态等。使用枚举可以避免魔法数字，提高代码可读性和维护性。

完整代码

/**
 * 订单状态枚举
 * 包含常见的订单生命周期状态
 */
public enum OrderStatus {
    PENDING("待支付", "用户已提交订单，等待支付"),
    PAID("已支付", "用户已完成支付"),
    SHIPPED("已发货", "商家已发货"),
    DELIVERED("已送达", "订单已送达用户"),
    CANCELLED("已取消", "订单已取消"),
    REFUNDED("已退款", "订单已退款");
    
    private final String displayName;
    private final String description;
    
    OrderStatus(String displayName, String description) {
        this.displayName = displayName;
        this.description = description;
    }
    
    public String getDisplayName() { return displayName; }
    public String getDescription() { return description; }
    
    public static OrderStatus fromDisplayName(String displayName) {
        for (OrderStatus status : values()) {
            if (status.displayName.equals(displayName)) {
                return status;
            }
        }
        return null;
    }
    
    public static void main(String[] args) {
        OrderStatus status = OrderStatus.PAID;
        System.out.println("状态名称: " + status.name());
        System.out.println("显示名称: " + status.getDisplayName());
        System.out.println("状态描述: " + status.getDescription());
        
        System.out.println("\n所有订单状态:");
        for (OrderStatus s : OrderStatus.values()) {
            System.out.println(s.ordinal() + ". " + s.name() + " - " + s.getDisplayName());
        }
    }
}

运行结果

状态名称: PAID
显示名称: 已支付
状态描述: 用户已完成支付

所有订单状态:
0. PENDING - 待支付
1. PAID - 已支付
2. SHIPPED - 已发货
3. DELIVERED - 已送达
4. CANCELLED - 已取消
5. REFUNDED - 已退款

场景2：策略模式（替换大量if/else）

应用场景

当需要根据不同条件执行不同算法时，可以使用枚举实现策略模式，避免冗长的if-else语句。

完整代码

/**
 * 折扣策略枚举
 * 使用枚举实现策略模式
 */
public enum DiscountStrategy {
    STUDENT("学生折扣", "适用于在校学生") {
        @Override
        public double applyDiscount(double price) {
            return price * 0.8; // 学生8折
        }
    },
    VIP("VIP折扣", "适用于VIP会员") {
        @Override
        public double applyDiscount(double price) {
            return price * 0.7; // VIP7折
        }
    },
    MEMBER("会员折扣", "适用于普通会员") {
        @Override
        public double applyDiscount(double price) {
            return price * 0.85; // 会员85折
        }
    },
    NORMAL("无折扣", "适用于普通用户") {
        @Override
        public double applyDiscount(double price) {
            return price;
        }
    };
    
    private final String strategyName;
    private final String description;
    
    DiscountStrategy(String strategyName, String description) {
        this.strategyName = strategyName;
        this.description = description;
    }
    
    public abstract double applyDiscount(double price);
    
    public String calculateFinalPrice(double price) {
        double finalPrice = applyDiscount(price);
        return String.format("原价: %.2f, 折扣后: %.2f, 节省: %.2f", 
                           price, finalPrice, price - finalPrice);
    }
    
    public static void main(String[] args) {
        double originalPrice = 100.0;
        for (DiscountStrategy strategy : DiscountStrategy.values()) {
            System.out.println("\n--- " + strategy.strategyName + " ---");
            System.out.println(strategy.calculateFinalPrice(originalPrice));
        }
    }
}

运行结果

--- 学生折扣 ---
原价: 100.00, 折扣后: 80.00, 节省: 20.00

--- VIP折扣 ---
原价: 100.00, 折扣后: 70.00, 节省: 30.00

--- 会员折扣 ---
原价: 100.00, 折扣后: 85.00, 节省: 15.00

--- 无折扣 ---
原价: 100.00, 折扣后: 100.00, 节省: 0.00

场景3：统一返回码（后端接口必备）

应用场景

在后端接口开发中，统一的返回码体系可以让前端更好地处理各种响应情况。

完整代码

public enum ResultCode {
    SUCCESS(200, "操作成功"),
    BAD_REQUEST(400, "请求参数错误"),
    UNAUTHORIZED(401, "未授权，请先登录"),
    FORBIDDEN(403, "禁止访问，权限不足"),
    NOT_FOUND(404, "资源未找到"),
    ERROR(500, "服务器内部错误");
    
    private final int code;
    private final String message;
    
    ResultCode(int code, String message) {
        this.code = code;
        this.message = message;
    }
    
    public int getCode() { return code; }
    public String getMessage() { return message; }
    public boolean isSuccess() { return this == SUCCESS; }
}
